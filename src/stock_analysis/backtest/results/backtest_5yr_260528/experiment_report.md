# 五年回测实验报告：四层加权排名系统预测准确性验证

> **实验日期**: 2026-05-28
> **实验目标**: 验证 InvestSkill v3.0 四层加权排名系统的历史预测准确性
> **实验标的**: 英伟达 (NVDA) · 小米 (1810.HK) · 特斯拉 (TSLA)
> **回测窗口**: 2021-05 至 2025-11（5年，每半年一个时点）

---

## 一、实验背景

### 1.1 研究目的

本实验旨在验证四层加权排名系统（Greenblatt v3.0）在历史时点的预测准确性，回答以下核心问题：

1. 系统给出的 BULLISH/NEUTRAL/BEARISH 信号是否能有效预测未来股价走势？
2. 概率加权预测与实际回报之间的校准度如何？
3. 不同公司、不同信号类型的预测效果是否存在显著差异？

### 1.2 排名系统概述

四层加权排名系统（InvestSkill v3.0）采用以下权重：

| 层级 | 维度 | 指标 | 权重 |
|------|------|------|:----:|
| L1 | 💰 便不便宜 | EBIT/EV | 40% |
| L2 | 🏭 赚不赚钱 | ROIC | 25% |
| L3 | 🛡️ 会不会崩 | Piotroski F-Score | 25% |
| L4 | 📈 增长值不值 | PEG | 10% |

**信号判定规则**：
- 排名前 1/3 → BULLISH（看多）
- 排名中间 1/3 → NEUTRAL（中性）
- 排名后 1/3 → BEARISH（看空）
- F-Score ≤ 3 → 强制 BEARISH

## 二、实验方法

### 2.1 数据隔离设计

为避免前瞻偏差（look-ahead bias），本实验采用严格的数据隔离机制：

```python
# 财报数据隔离：仅使用截止日之前90天已公开的财报
avail_dt = cutoff_dt - pd.Timedelta(days=90)

# 价格数据隔离：仅使用截止日之前的收盘价
hist = yf.download(ticker, start=start, end=cutoff_str)
close = hist.loc[hist.index <= cutoff].iloc[-1]
```

### 2.3 概率加权预测模型

基于信号类型和排名位置计算上涨概率和预期回报：

```python
# 基础概率
BULLISH: prob_up = 60%, expected_return = +15%
NEUTRAL: prob_up = 50%, expected_return = +5%
BEARISH: prob_up = 40%, expected_return = -10%

# 排名调整
rank_adjustment = 1.0 - (rank_num / total)
prob_up += (rank_adjustment - 0.5) * 0.2  # ±10%调整
expected_return += rank_adjustment * 0.1    # ±10%调整
```

### 2.4 评估指标

| 指标 | 定义 |
|------|------|
| **方向正确率** | 预测方向与实际方向一致的比例 |
| **概率校准度** | 1 - \|预测概率 - 实际正确率\| |
| **回报误差** | \|预期回报 - 实际回报\| |

## 五、核心发现与局限性

### 5.1 核心发现

| 发现 | 证据 | 可信度 |
|------|------|:------:|
| BULLISH 信号可靠 | 校准度 95.3%，正确率 60% | ⭐⭐⭐⭐ |
| BEARISH 信号失效 | 正确率仅 12.5% | ⭐⭐⭐⭐ |
| 英伟达预测最佳 | 正确率 50%，平均回报 +36.56% | ⭐⭐⭐ |
| 特斯拉预测最差 | 正确率 16.7% | ⭐⭐⭐ |
| 系统倾向保守 | 多数时点给出 NEUTRAL/BEARISH | ⭐⭐⭐ |

### 5.2 系统偏差分析

**保守偏差（Conservative Bias）**：
- 系统过于频繁地给出 BEARISH 信号（44.4% 的时点）
- 实际结果显示，BEARISH 信号下仍有 87.5% 的时点上涨
- 原因：估值指标（EBIT/EV、PE）对高增长科技股天然偏贵

**高增长股失效**：
- 英伟达、特斯拉等高增长股的估值长期偏高
- 系统将"偏贵"解读为"看空"，但市场持续给予增长溢价
- F-Score 对科技股区分度不足（多数为 3/9）

### 5.3 数据局限性

| 局限 | 影响 | 解决方案 |
|------|------|---------|
| yfinance 财报仅 3-4 年 | 无法覆盖 5 年回测 | 使用 SEC EDGAR 原始数据 |
| 小米部分时点 NaN | 降低统计可信度 | 增加数据验证逻辑 |
| 未考虑交易成本 | 高估实际回报 | 加入滑点和手续费 |

## 六、改进建议

### 6.1 信号判定优化

```python
# 当前规则（过于简单）
if rank_num <= total // 3:
    signal = "BULLISH"

# 建议改进（考虑增长溢价）
if rank_num <= total // 3:
    if growth_rate > 0.20 and f_score >= 5:
        signal = "BULLISH"  # 高增长 + 安全 → 看多
    elif pe_ratio > 100:
        signal = "NEUTRAL"  # 极高估值 → 降级
    else:
        signal = "BULLISH"
```

### 6.2 权重动态调整

针对不同行业/市场特征，动态调整四层权重：

| 市场特征 | L1 权重 | L2 权重 | L3 权重 | L4 权重 |
|---------|:------:|:------:|:------:|:------:|
| 价值股 | 50% | 20% | 20% | 10% |
| 成长股 | 25% | 25% | 20% | 30% |
| 周期股 | 40% | 30% | 20% | 10% |

### 6.3 数据源升级

| 数据源 | 覆盖范围 | 优势 |
|--------|---------|------|
| SEC EDGAR | 10+ 年 | 原始 XBRL 数据 |
| Alpha Vantage | 20+ 年 | 标准化财务数据 |
| Financial Modeling Prep | 10+ 年 | 实时更新 |
